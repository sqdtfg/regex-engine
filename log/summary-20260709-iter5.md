# 迭代5 — 修复嵌套括号回归

## 修改的问题

**根因**: 迭代3 中在 `parse_atom` 添加了 `)` 视为普通字符的 hack，导致内部括号匹配的 `expect(RPAREN)` 无法消费分组的 `)` — 因为 `)` 被 `parse_atom` 截胡。

**修复方法**: 
1. 从 `parse_atom` 移除 `)` 当作普通字符的特例
2. 从 `parse_chain` 中移除 `TOK_RPAREN` 从 `is_atom_start`
3. 保留 `parser_parse` 中对结尾 `)` 的 ERE 兼容（直接忽略）
4. 更新 `test_parser.c` 中 `a)` 的期望值

## 解决的问题

- `(a|b)` 等所有嵌套括号解析正常
- `a(b)c`、`a(b|c)*d`、`(a)+(b)?` 等全部通过
- 全部 9 个内部测试套件 676+ 项全部通过
- 无回归

## ERE 测试结果

|------|-----|
