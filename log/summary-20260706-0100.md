# 2026-07-06 01:00 — 迭代 2：捕获组崩溃修复 + 贪婪匹配

## 修改的问题

### 1. capture.c: use-after-free 导致崩溃
- **根因**：`dfa_from_ast_with_groups` 保存了 `ASTNode*` 指针到 `GroupMeta.sub_ast`，但测试代码在构建 DFA 后立即 `ast_free(ast)` 释放了原始 AST。后续 `dfa_match_captured` 调用 `match_sub_dfa_greedy(meta->sub_ast, ...)` 时访问了已释放的内存。
- **修复**：
  - 新增 `ast_clone()` 函数（[parser.c](src/parser.c) + [parser.h](include/parser.h)）
  - `collect_group_metas_recursive` 改为克隆子 AST 而非保存原始指针
  - `dfa_capture_free` 中释放所有克隆的子 AST
  - `GroupMeta.sub_ast` 从 `const ASTNode*` 改为 `ASTNode*`

### 2. capture.c: 最短匹配 vs 贪婪匹配
- **问题**：`dfa_match_captured` 调用 `dfa_match` 获取完整匹配区间，但 `dfa_match` 返回**最短**匹配（第一个接受状态）。对于 `(a+)` 在 "xxxaaaayyy" 上，只匹配到单个 'a'，导致捕获组只捕获 "a" 而非 "aaa"。
- **修复**：在 `dfa_match_captured` 内联实现贪婪扫描逻辑——从每个起始位置扫描整个输入，记录最后一个接受状态的位置。
